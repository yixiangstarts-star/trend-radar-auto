# 📋 明天操作清单 —— 科技新闻采集系统 MVP 上线

> 预估总耗时：**约 1.5-2 小时**
> 分成 **6 步**，按顺序执行，每步都有验证方法。

---

## 📌 开始前确认

在开始之前，先确认以下条件：

- [x] 有 **GitHub 账号**（没有的话去 github.com 注册，2 分钟）
- [x] 有 **腾讯 IMA 账号**（就是你平时用的那个）
- [x] 电脑上装了 **Python 3.11+**（打开终端输入 `python --version` 检查）

---

## 第一步：申请 IMA OpenAPI 凭证 ⏱️ ~10 分钟

**做什么**：获取 API 密钥，让脚本能自动写笔记到你的 IMA 知识库。

1. 打开浏览器，访问 **https://ima.qq.com/agent-interface**
2. 登录你的 IMA 账号
3. 点击「创建应用」或类似按钮
4. 记下生成的 **Client ID** 和 **API Key**（⚠️ 重要！保存好，只显示一次）

**验证方式**：拿到两个值的字符串，类似 `client_abc123...` 和 `key_xyz789...`

---

## 第二步：在 IMA 中创建分类笔记 ⏱️ ~10 分钟

**做什么**：建立 8 个分类笔记，作为日报的分类存储目标。

1. 打开 IMA 客户端或网页版
2. 在你的知识库（或默认笔记本）中，按以下顺序创建 8 个笔记：

| 笔记标题 | 分类代码 | 内容（初始） |
|----------|:--------:|------|
| C01-AI与大模型 | C01 | `# AI与大模型` |
| C02-芯片与半导体 | C02 | `# 芯片与半导体` |
| C03-智能汽车与新能源 | C03 | `# 智能汽车与新能源` |
| C04-互联网平台与产品 | C04 | `# 互联网平台与产品` |
| C05-创投与融资 | C05 | `# 创投与融资` |
| C06-硬件与消费电子 | C06 | `# 硬件与消费电子` |
| C07-科技政策与监管 | C07 | `# 科技政策与监管` |
| C08-开源与技术工具 | C08 | `# 开源与技术工具` |

3. 记下每个笔记的 **note_id**（在笔记详情或URL中查找）

> 💡 如果 IMA 不显示 note_id，可以先不做这步，等后续 IMA 客户端有搜索功能后再配置。

**验证方式**：8 个笔记已创建，能看到它们在你的笔记本里。

---

## 第三步：配置项目凭证 ⏱️ ~5 分钟

**做什么**：把第一步拿到的凭证填入配置文件。

1. 用记事本或 VS Code 打开 `tech-news-pipeline/sources.yaml`
2. 找到 `ima:` 下面的这段：

```yaml
ima:
  client_id: "YOUR_CLIENT_ID"    # ← 改成你的
  api_key: "YOUR_API_KEY"        # ← 改成你的
```

3. 替换为你的真实凭证

**⚠️ 安全提醒**：这个文件包含密钥，**不要上传到公开的 GitHub 仓库**！
（后面会在 GitHub Secrets 中设置，不用担心。）

**做什么**：把代码上传到 GitHub，让 Actions 每天自动跑。


## 第六步：配置 GitHub Secrets + 验证定时任务 ⏱️ ~15 分钟

**做什么**：在 GitHub 中安全存储你的 IMA 凭证，然后手动触发一次测试。

1. 打开你的 GitHub 仓库 → **Settings** → **Secrets and variables** → **Actions**
2. 点击 **New repository secret**，依次添加：

| Secret 名称 | 值 |
|-------------|-----|
| `IMA_CLIENT_ID` | 你的 Client ID |
| `IMA_API_KEY` | 你的 API Key |
| `IMA_KB_ID` | 你的知识库 ID（可选） |
| `WECHAT_WEBHOOK` | 企业微信 Webhook URL（可选） |

3. 回到仓库 → **Actions** 标签页
4. 找到 **Daily Tech News Collection** → **Run workflow**
5. 勾选 **dry_run: true**（第一次用试运行模式），点击 **Run workflow**
6. 等待 2-3 分钟，观察运行日志
7. 成功后，去 Actions → 该次 run → Artifacts 下载日报 .md 查看

**验证方式**：
- Actions 日志中没有红色报错
- Artifact 里有日报文件可以下载
- `dry_run` 模式下本地也生成了 output/*.md

## ❓ 常见问题

**Q: 本地试运行报错 "No module named 'xxx'"？**
A: 确保先执行 `pip install -r requirements.txt`

**Q: IMA 写入失败？**
A: 检查 Client ID 和 API Key 是否正确。可以用 `--dry-run` 先跳过 IMA。

**Q: GitHub Actions 没按时间触发？**
A: GitHub Actions 的 cron 可能有 5-10 分钟延迟，这是正常的。手动触发可以立即测试。

**Q: 想把项目设成公开仓库？**
A: **不建议**。配置文件里有 IMA 凭证，即使用了 Secrets，sources.yaml 里还有 IMA base_url 等配置。保持 Private 更安全。
